<datalist>

Define uma lista de opções predefinidas para um campo <input>, permitindo que o usuário selecione uma das opções sugeridas ao digitar no campo.

Este é o Combo Box ele dá uma lista de opções, onde só é permitido escolher entre as opções disponíveis. Não permite digitar, quando tem muitas opções fica mais difícil a navegação. Ele permite que ao digitar uma letra ele vai para a primeira opção que comece com a letra.

O Data List oferece sugestões para preencher o campo. é interessante quando se quer utilizar buscas sugeridas.

<code>, <kbd>, <pre>

Tags voltadas para colocar comandos ou código fonte dentro da página HTML. Dá uma formatação pré-definida do navegador.

<code> é usado para mostrar código fonte, como HTML, CSS, JavaScript, PHP, etc. O código é mostrado com uma fonte monoespaçada.
Exemplo: Para trocar o fundo de um elemento, utilize o comando background-color.

<kbd> é usado para mostrar teclas no teclado. O texto é em itálico e não é formatado como código.
Exemplo: Para copiar esta informação, pressione Ctrl+C.

<><pre> é usado para mostrar código fonte, mas não dá a característica de fonte monoespaçada. O código é mostrado com uma fonte padrão do navegador.
Exemplo: Comando completo para alerar a cor de fundo:

            
// Código para mudar a cor de fundo da tag code.

           
            code{
              background-color: f1f1f1;
              color: #801a1a;
            }

<details>, <sumary>

<details> cria um widget interativo que o usuário pode abrir ou fechar.é usado para mostrar uma lista de detalhes sobre um item. Ele é mais útil quando há muitos detalhes para mostrar. Ideal para seções de conteúdo que podem ser ocultas, como em uma lista de produtos e FAQs.

<sumary> define um título ou legenda para o elemento <details>. Esse título é o que aparece na interface e pode ser clicado para abrir/fechar o <details>.


Exemplo: Uma lista de produtos com detalhes:

<meter>, <progress>

Os elementos <meter> e <progress> são usados para mostrar valores numéricos, como progresso de uma tarefa, porcentagem, etc.

<meter> é um medidor, representa uma medição escalar dentro de um intervalo conhecido. Utilizado para exibir valores que têm um intervalo máximo e mínimo, como uma barra de progresso numérica.

<mark> (com Javascript)

Realça o texto dentro da tag com uma cor de fundo. Com JavaScript, pode ser usada para destacar dinamicamente partes do texto, como em uma pesquisa dentro da página.

Novos atributos bem úteis no HTML5

  • form permite que você especifique a qual formulário um elemento deve se associar. O valor desse atributo deve ser o id do formulário ao qual o controle está vinculado.

  • autofocus é usado para especificar que um elemento deve receber automaticamente o foco quando a página carrega.

    Apenas um elemento em uma página pode ter o autofocus. Se houver mais de um elemento com autofocus, o primeiro no código será focado.

    Quando a página é carregada ou o elemento é inserido no DOM, o navegador automaticamente coloca o cursor ou foco de interação no elemento que tem o autofocus. Isso é útil para melhorar a experiência do usuário, direcionando imediatamente sua atenção para um campo específico.

  • placeholder é usado para fornecer um texto de dica dentro de um campo de formulário (como %ltinput> ou <textarea>) que desaparece quando o usuário começa a digitar no campo.

    Embora útil, o placeholder não deve substituir um rótulo (<label>), especialmente por razões de acessibilidade, pois alguns leitores de tela não reconhecem o placeholder como um rótulo.

  • required é usado para indicar que um campo de formulário é obrigatório e deve ser preenchido antes de o formulário ser enviado.

    Se um campo que possui o atributo required estiver vazio no momento do envio do formulário, o navegador impedirá o envio e alertará o usuário de que o campo precisa ser preenchido.

    A validação é feita automaticamente pelo navegador, sem a necessidade de JavaScript adicional. Se o campo não for preenchido, uma mensagem de erro padrão é exibida, e o foco é colocado no campo obrigatório.

    Embora o required faça parte da validação nativa do HTML5, as mensagens de erro padrão podem ser personalizadas com JavaScript, se necessário.

  • disabled quando um elemento possui o atributo disabled, ele não pode ser clicado, editado ou selecionado pelo usuário. Além disso, o navegador não envia o valor desse elemento quando o formulário é submetido.

    Elementos disabled geralmente são exibidos com um estilo visual diferenciado, como um texto cinza ou esmaecido, para indicar ao usuário que estão inativos.

    Não é possível interagir com elementos desativados, nem mesmo com JavaScript (por exemplo, onclick). Contudo, o atributo disabled pode ser ativado ou desativado dinamicamente usando JavaScript.

  • minlength e maxlength definem o número mínimo e máximo de caracteres que o usuário deve inserir em um campo de texto para que o formulário seja considerado válido.
